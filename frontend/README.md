# Infinite ERP - Frontend Application\n\n## Folder Structure\n\n```\nsrc/\n├── components/           # Reusable UI components\n│   ├── common/          # Common components (buttons, inputs, etc.)\n│   ├── forms/           # Form-specific components\n│   └── tables/          # Table-specific components\n├── constants/           # Application constants\n│   └── permissions.js   # Permission constants and routes\n├── hooks/              # Custom React hooks\n│   └── usePermissions.js # Permission checking hook\n├── layouts/            # Layout components\n│   ├── AppLayout.jsx   # Main application layout\n│   ├── AppHeader.jsx   # Header component\n│   └── AppSidebar.jsx  # Sidebar component\n├── pages/              # Page components\n│   ├── auth/           # Authentication pages\n│   │   └── Login.jsx   # Login page\n│   ├── dashboard/      # Dashboard pages\n│   │   └── Dashboard.jsx\n│   ├── masters/        # Master data pages\n│   │   ├── UserMaster.jsx\n│   │   ├── UserForm.jsx\n│   │   ├── ConcernMaster.jsx\n│   │   └── PartyMaster.jsx\n│   ├── transactions/   # Transaction pages\n│   ├── reports/        # Report pages\n│   └── settings/       # Settings pages\n├── routes/             # Route configuration\n│   └── ProtectedRoute.jsx\n├── services/           # API services\n│   ├── api.js          # Axios configuration\n│   └── authService.js  # Authentication API calls\n├── store/              # Redux store\n│   ├── index.js        # Store configuration\n│   └── slices/         # Redux slices\n│       ├── authSlice.js\n│       └── uiSlice.js\n└── utils/              # Utility functions\n    └── helpers.js      # Common helper functions\n```\n\n## Key Features Implemented\n\n### 1. Authentication System\n- Login page with username/password\n- JWT token management\n- Protected routes\n- User session management\n\n### 2. Permission-Based UI\n- Checkbox-based permission model\n- Admin user override\n- Dynamic menu visibility\n- Action button control (Add/Edit/Delete)\n\n### 3. Layout Components\n- Responsive Ant Design layout\n- Collapsible sidebar\n- Header with user info and logout\n- Professional ERP styling\n\n### 4. Master Data Management\n- User Master (Login Creation)\n- Concern Master with tabs\n- Party Master (Supplier/Customer)\n- Form validation and error handling\n\n### 5. State Management\n- Redux Toolkit for state management\n- Separate slices for auth and UI\n- Centralized permission checking\n\n## Permission Model\n\nEach user has the following boolean permissions:\n- `adminUser`: Full access override\n- `dcClose`: Access to DC Close functionality\n- `active`: User account status\n- `add`: Can create new records\n- `edit`: Can modify existing records\n- `delete`: Can delete records\n\n## Usage Instructions\n\n### 1.
